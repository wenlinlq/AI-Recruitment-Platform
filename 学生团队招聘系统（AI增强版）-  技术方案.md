## 三、MongoDB数据模型设计

### 核心集合结构

| 集合名              | 主要字段                                                     | 索引策略                                            |
| ------------------- | ------------------------------------------------------------ | --------------------------------------------------- |
| **users**           | _id, username, password, email, role, avatar, status, createdAt | username唯一索引、email唯一索引                     |
| **studentProfiles** | _id, userId, realName, studentId, major, grade, skills, projects, resumeUrl, resumeParseResult | userId唯一索引、skills多键索引                      |
| **teams**           | _id, name, department, description, leaderId, members, contact, avatar | name索引、department索引                            |
| **positions**       | _id, teamId, title, type, requirements, responsibilities, status, deadline, aiAnalysis | teamId索引、全文索引(title+description)、status索引 |
| **applications**    | _id, positionId, studentId, resumeUrl, status, aiMatch, interviews, feedback | 复合唯一索引(positionId+studentId)、status索引      |
| **aiConversations** | _id, userId, type, context, messages, summary, createdAt     | userId索引、type索引                                |

### 关键设计说明

1. **文档嵌套**：利用MongoDB文档型特点，将相关数据内嵌（如学生项目经历、AI解析结果）
2. **灵活模式**：AI分析结果使用Mixed类型，适应不同模型的输出格式
3. **全文索引**：职位标题和描述建立全文索引，支持关键词搜索
4. **地理空间**：如需定位功能，可添加location字段支持地理查询

## 八、核心优势总结

### Node.js + MongoDB 方案优势

1. **全栈JavaScript**：前后端语言统一，开发效率高
2. **MongoDB灵活模式**：适应AI结果的非结构化数据
3. **流式处理友好**：Node.js天然支持SSE/WebSocket
4. **快速迭代**：无需ORM映射，Mongoose提供便捷API
5. **生态丰富**：NPM上有大量AI相关库可直接使用

### AI增强价值

- 简历处理时间从**10分钟**缩短到**10秒**
- 筛选效率提升**5-10倍**
- 面试准备时间减少**70%**
- 招聘决策数据化、智能化

## 九、注意事项

1. **MongoDB设计要点**
   - 合理使用嵌套文档，避免过深嵌套
   - 建立合适的复合索引
   - 考虑数据增长的分片策略
2. **AI成本控制**
   - 缓存重复的AI请求
   - 使用embedding本地计算相似度
   - 长文本分片处理
3. **安全考虑**
   - JWT过期策略
   - 文件上传类型限制
   - API限流防刷
   - MongoDB注入防护
